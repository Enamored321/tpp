# Стандарти та інструменти документування проєкту

Для забезпечення високої якості коду та полегшення його підтримки в проєкті впроваджено сучасні стандарти документування.

## Стандарти документування

1.  **JavaScript (JSDoc)**:
    - Використовується формат спеціальних коментарів `/** ... */`.
    - Обов’язково описуються параметри (`@param`), типи повернених значень (`@returns`) та призначення функцій.
    - Стандарт JSDoc є галузевим стандартом для JS, що дозволяє IDE (наприклад, VS Code) надавати підказки під час написання коду.

2.  **HTML/CSS**:
    - Використовуються змістовні секційні коментарі для відокремлення логічних блоків.
    - Для CSS документуються змінні (Custom Properties) та призначення основних класів.

## Інструменти автоматичної генерації

1.  **JSDoc**:
    - Основний інструмент для генерації HTML-документації з JSDoc-коментарів.
    - Підтримує плагіни для покращення вигляду (наприклад, `docdash`).

2.  **jsdoc-to-markdown**:
    - Утиліта для перетворення JSDoc у Markdown-файли, що зручно для інтеграції прямо в репозиторій (наприклад, у теці `docs/`).

3.  **Doxygen**:
    - Універсальний інструмент для багатьох мов (C++, Java, JS). Генерує структуровану документацію з перехресними посиланнями.

4.  **Storybook**:
    - Для проєктів з компонентним підходом (UI-бібліотеки). Дозволяє документувати та тестувати візуальні компоненти в ізоляції.

## Як документувати нові функції

Кожна нова функція повинна мати блок коментарів наступного вигляду:

```javascript
/**
 * Короткий опис того, що робить функція.
 * @param {Type} paramName - Опис параметра.
 * @returns {Type} Опис того, що повертає функція.
 */
function myNewFunction(paramName) {
    // ...
}
```
