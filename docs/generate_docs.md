# Інструкція з генерації документації проєкту

Для автоматичної генерації технічної документації в проєкті використовується **JSDoc**.

## Вимоги

- Встановлений **Node.js** (версія 16+).
- Виконана установка залежностей через `npm install`.

## Як згенерувати документацію

Для запуску процесу генераії виконайте наступну команду в кореневій директорії проєкту:

```bash
npm run docs:gen
```

## Результат генерації

- Документація буде згенерована у форматі HTML у директорії `docs/gen/`.
- Головний файл документації: `docs/gen/index.html`.
- Усі публічні функції та методи з файлу `js/main.js` автоматично потраплять до звіту завдяки JSDoc-коментарям.

## Конфігурація

Налаштування інструменту знаходяться у файлі `jsdoc.json`. Там можна змінити:
- Шляхи до вихідних файлів (`source.include`).
- Директорію для результатів (`opts.destination`).
- Використовувані плагіни та шаблони.

## Архівування (для здачі)

Щоб створити архів із документацією для передачі, можна скористатися командою:
```powershell
Compress-Archive -Path docs/gen/* -DestinationPath project_docs.zip -Force
```
Або просто заархівувати теку `docs/gen/` будь-яким доступним архіватором.
