# Налаштування статичного аналізу та лінтингу

У проєкті використовуються сучасні інструменти для забезпечення високої якості коду, дотримання єдиного стилю та запобігання поширеним помилкам.

## Обрані інструменти

1.  **ESLint** (JavaScript):
    - **Чому:** Промисловий стандарт для аналізу JavaScript. Дозволяє виявляти помилки логіки, потенційні проблеми з безпекою та контролювати стиль написання коду.
    - **Версія:** 10.1.0+ (з використанням Flat Config).

2.  **Stylelint** (CSS):
    - **Чому:** Найпотужніший інструмент для перевірки CSS-файлів. Запобігає дублюванню селекторів, неправильному використанню властивостей та забезпечує консистентність стилів.

3.  **Prettier**:
    - **Чому:** Автоматичне форматування коду. Знімає потребу в ручному вирівнюванні відступів та лапок, забезпечуючи однаковий вигляд файлів у всієї команди.

4.  **HTML Validate**:
    - **Чому:** Перевірка валідності HTML5 структури та дотримання стандартів семантики та доступності.

## Базові правила та налаштування

### ESLint (`eslint.config.js`)

- **Стиль:** 4 пробіли для відступів, одинарні лапки (`'`), обов'язкові крапки з комою (`;`).
- **Якість:** Заборона неикористаних змінних (`no-unused-vars`), суворе порівняння (`eqeqeq`), заборона використання `var` (`no-var`), обов'язкове використання `const` для незмінних змінних (`prefer-const`).
- **Безпека:** Заборона `eval()` та `no-script-url` для запобігання XSS-атакам.

### Stylelint (`.stylelintrc.json`)

- Використовується стандартний конфіг `stylelint-config-standard`.
- Дозволено довільний формат імен класів (`selector-class-pattern: null`) для гнучкості.

### Prettier (`.prettierrc`)

- Налаштована ширина рядка (100 символів), 4 пробіли, використання одинарних лапок та обов'язкові коми в кінці об'єктів для покращення диференціації в Git.

## Інструкція з запуску

Усі інструменти налаштовані як NPM-скрипти у `package.json`:

1.  **Повна перевірка проєкту:**

    ```bash
    npm run lint
    ```

    Ця команда послідовно запускає лінтинг для JS, CSS та HTML.

2.  **Перевірка окремих частин:**
    - `npm run lint:js` — тільки JavaScript.
    - `npm run lint:css` — тільки CSS.
    - `npm run lint:html` — тільки HTML.

3.  **Автоматичне виправлення (форматування):**
    ```bash
    npm run format
    ```
    Виправляє відступи, лапки та переноси рядків у всьому проєкті.

## Процес прийому коду (Linting Flow)

Перед кожним комітом рекомендується запускати `npm run format` та `npm run lint`, щоб переконатися, що код відповідає стандартам проєкту. Це зменшує час на рев'ю та запобігає появі технічного боргу.
