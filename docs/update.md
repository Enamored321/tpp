# Інструкція з оновлення системи (Update Procedure)

Ця документація описує процес безпечного оновлення статичного вебзастосунку на сервері (production) або в контейнері.

## 1. Планування технологічного вікна
Оскільки архітектура застосунку статична (HTML/CSS/JS, Vite), оновлення відбувається майже миттєво. Рекомендоване "технологічне вікно" для оновлення — в години найменшого навантаження користувачів (наприклад, 02:00 - 05:00), хоча зміна файлів вебсервером Nginx відбувається без переривання сервісу (Zero-Downtime Deployment).

## 2. Підготовка до оновлення
1. Переконайтеся, що всі зміни злиті (merged) у гілку `main`.
2. Переконайтесь у проходженні всіх автоматизованих перевірок якості: 
   ```bash
   npm run lint
   ```
3. Створіть резервну копію діючої збірки (забезпечує швидкий відкат):
   ```bash
   sudo cp -r /var/www/html/tpp /var/www/html/tpp_backup_$(date +%F)
   ```

## 3. Процес оновлення
Оновлення можна виконати кількома способами, залежно від інфраструктури.

### Спосіб А: Ручне оновлення коду (Заміна скомпільованих версій)
1. Перейдіть у тимчасову теку (або папку репозиторію) та завантажте нову версію коду:
   ```bash
   cd /tmp/tpp
   git pull origin main
   ```
2. Встановіть залежності та зберіть нову статику:
   ```bash
   npm install
   npm run build
   ```
3. Замініть існуючі файли на вебсервері новими артефактами (використовуємо rsync для чистої заміни):
   ```bash
   sudo rsync -av --delete dist/ /var/www/html/tpp/
   ```
4. Перезавантажте Nginx безпечним способом `reload` без розриву поточних з'єднань:
   ```bash
   sudo systemctl reload nginx
   ```

### Спосіб Б: Оновлення через Docker (Контейнеризація)
1. Стягніть репозиторій та зберіть новий образ докеру:
   ```bash
   docker build -t tpp-landing:latest .
   ```
2. Зупиніть старий контейнер та запустіть новий:
   ```bash
   docker stop web-landing
   docker rm web-landing
   docker run -d -p 80:80 --name web-landing tpp-landing:latest
   ```

## 4. Швидкий відкат (Rollback Procedure)
Якщо після оновлення система поводиться нестабільно:

**Для способу локальної заміни файлів:**
Відновіть артефакти з бекапу:
```bash
sudo rsync -av --delete /var/www/html/tpp_backup_$(date +%F)/ /var/www/html/tpp/
sudo systemctl reload nginx
```

**Для Docker:**
Запустіть попередній збережений тег образу:
```bash
docker stop web-landing
docker rm web-landing
docker run -d -p 80:80 --name web-landing tpp-landing:previous_stable
```
